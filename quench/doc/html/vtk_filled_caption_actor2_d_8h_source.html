00018 // .NAME vtkFilledCaptionActor2D - draw text label associated with a point
00019 // .SECTION Description
00020 // vtkFilledCaptionActor2D is a hybrid 2D/3D actor that is used to associate text 
00021 // with a point (the AttachmentPoint) in the scene. The caption can be 
00022 // drawn with a rectangular border and a leader connecting 
00023 // the caption to the attachment point. Optionally, the leader can be 
00024 // glyphed at its endpoint to create arrow heads or other indicators.
00025 // 
00026 // To use the caption actor, you normally specify the Position and Position2
00027 // coordinates (these are inherited from the vtkActor2D superclass). (Note
00028 // that Position2 can be set using vtkActor2D's SetWidth() and SetHeight()
00029 // methods.)  Position and Position2 define the size of the caption, and a
00030 // third point, the AttachmentPoint, defines a point that the caption is
00031 // associated with.  You must also define the caption text, 
00032 // whether you want a border around the caption, and whether you want a
00033 // leader from the caption to the attachment point. The font attributes of 
00034 // the text can be set through the vtkTextProperty associated to this actor.
00035 // You also indicate whether you want
00036 // the leader to be 2D or 3D. (2D leaders are always drawn over the
00037 // underlying geometry. 3D leaders may be occluded by the geometry.) The
00038 // leader may also be terminated by an optional glyph (e.g., arrow).
00039 //
00040 // The trickiest part about using this class is setting Position, Position2,
00041 // and AttachmentPoint correctly. These instance variables are
00042 // vtkCoordinates, and can be set up in various ways. In default usage, the
00043 // AttachmentPoint is defined in the world coordinate system, Position is the
00044 // lower-left corner of the caption and relative to AttachmentPoint (defined
00045 // in display coordaintes, i.e., pixels), and Position2 is relative to
00046 // Position and is the upper-right corner (also in display
00047 // coordinates). However, the user has full control over the coordinates, and
00048 // can do things like place the caption in a fixed position in the renderer,
00049 // with the leader moving with the AttachmentPoint.
00050
