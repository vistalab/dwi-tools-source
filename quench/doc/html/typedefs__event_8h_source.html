00004 /* --------------------------------- Event Handling ---------------------------------- 
00005  * Event handling in Quench is done by event listener and source idiom.
00006  * A event source maintains a list of all the listeners for an event. 
00007  * When the event occurs the event source notifies all the listeners about the event.
00008  *
00009  * Events are of two types:
00010  * Simple events: Like enums
00011  * Events with data. All the event class implement a GetValue method which retrieves
00012  * the value associated with the event
00013  *
00014  * Events are used to exchange information among GUI elements. The datamodel is mostly
00015  * oblivious of events and acts like a structure with a few operation
00016  /* ----------------------------------------------------------------------------------- */
